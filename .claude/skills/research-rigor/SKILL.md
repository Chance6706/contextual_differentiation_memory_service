# Research Rigor — the CDMS experiment protocol

Distilled from what this project already does the hard way, plus the best of two community skills.
Core stance: **make exhaustively-tested, FALSIFIABLE assertions** — not "definitive" proofs, not merely
descriptive claims. Falsifiability is the boundary-drawing tool. A measurement is faithful + auditable:
you record results **and** deviations, never silently change what a number means.

## The protocol (in order)

1. **Pre-register before you run.** Write down, and freeze, BEFORE data exists: the estimand, the ONE primary
   endpoint, the gates, the decision rule, and the confirmatory-vs-exploratory status. A locked spec + a
   `--confirmatory` code path beats "we'll decide after seeing it." (This repo: `docs/validation/.../*LOCK.md`,
   `*PREREG.md`.)

2. **Smallest trustworthy target first.** Before the full grid, run the minimal honest version — a smoke test
   (a few calls) or a mini-pilot (small model subset × facet sample). It catches driver bugs, cost surprises,
   and design failures cheaply. Scale only once the small target is clean. (RigorPilot: inference < eval <
   training; here: smoke → mini-pilot → full matrix.)

3. **Pressure-test before locking (CLAUDE.md #12).** For any artifact hard to change later (pre-reg, builder/test
   pair, adapter contract, critical config), run BOTH lenses: (a) red-team — "how could this mislead, fail
   silently, be bypassed, or break on Python/network/disk quirks?" and (b) legitimate-use — "where does this
   over-design or have rough ergonomics?" Apply MUST_FIX/SHOULD_FIX inline; record inherent limits on the
   artifact itself. This has caught real bugs here (NaN cap bypass, CRLF header smuggling, cache collisions,
   pseudoreplication).

4. **Fresh cache for re-runs (CLAUDE.md #13).** Deliberate re-runs use a fresh timestamped cache dir. Crash-resume
   is the ONLY sanctioned cross-run cache reuse; publication needs a fresh end-to-end pass.

5. **Inference discipline.**
   - **Avoid pseudoreplication** — the #1 fatal error. The replicate is the unit the treatment is randomized
     over (facet/model), NOT each response. Resample at the cluster level (cluster/triplet bootstrap). Pooling
     responses as independent inflates significance (this repo's response-pooled z=+6.5 artifact).
   - **Effect sizes + CIs, not p alone.** Report the estimate, a one-sided LB / CI, and a permutation p.
   - **Equivalence, not just difference,** when the claim is "no meaningful gap" (e.g. surfacing parity): a real
     TOST-equivalent CI ⊂ ±δ, not a lenient point check.
   - **No post-hoc/observed power.** Use an a-priori MDE / sensitivity range. State the SESOI — the smallest
     effect that would change a conclusion — and *why* (not a bare Cohen convention).

6. **Multiple comparisons across cells — decide the frame, pre-committed.** As a grid grows (levels × models ×
   arms), pick ONE and write it down before analysis:
   - **Replication frame** (usual here): each arm is an independent replication of one hypothesis. Evidence =
     consistency, via a pre-committed **K/M-confirm rule** (e.g. LB>0 in ≥⌈2/3·M⌉ arms). Do NOT Bonferroni
     independent replications.
   - **Family frame:** the cells are separate hypotheses → correct (Bonferroni/FDR) or label the extras
     exploratory.
   Either way: **report every cell's numbers** (no cherry-picking), and route cells that fail a *precondition*
   (e.g. inadequate surfacing → undefined lift) to the precondition analysis, NOT to "failed to confirm."

7. **No HARKing.** A hypothesis generated by looking at the data (a smoke-test surprise, a subgroup that popped)
   is EXPLORATORY / hypothesis-generating — label it so. Confirming it needs a fresh pre-registered run with new
   data/seeds. A clean design does not upgrade an after-the-fact hypothesis to confirmatory.

8. **Blind what can bias.** Keep the judge/scorer blind to condition (e.g. identical prompt across REAL/DECOY;
   the judge never sees the manipulation). Blinding kills observer/detection bias.

9. **Report at audit grade.** Distinguish **verified / partial / blocked** states. Record every departure from a
   pure derivation or borrowed term (CLAUDE.md #11 → `docs/DEVIATIONS.md`) and every anomaly/open question with
   status + n. State comparability vs the baseline. **Completeness:** default to the FULL design; make any
   cost/scope cut EXPLICIT and the user's call — never a silent trim, and log every dropped cell.

## Bias & fallacy checklist (scan conclusions against these)

Confirmation bias · **HARKing / outcome-switching** · p-hacking / subgroup fishing · cherry-picked reporting ·
correlation→causation · base-rate neglect · **uncorrected multiple comparisons** · overgeneralization beyond the
sample · conflating statistical significance with practical importance · **unfalsifiability** (trained-in ≠ true;
a claim true-by-construction is not a behavioral finding). Name the specific flaw, say what evidence would fix it.
